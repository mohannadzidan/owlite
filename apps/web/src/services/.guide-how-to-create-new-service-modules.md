# Services Guide

## Overview

The `services/` directory contains API clients. Each module wraps one domain boundary and exposes grouped namespace exports — no classes, no singletons, just plain objects of functions.

---

Import `request` directly from `./request` and call the app's own `/api/...` routes.

```ts
import { request } from "./request";

export const articles = {
  list: () => request<Article[]>("/api/articles"),
  create: (body: NewArticle) =>
    request<Article>("/api/articles", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(body),
    }),
};
```

###hit external APIs

When the external API requires shared config (base URL, auth headers), declare a module-level `request` constant that wraps the utility with those defaults, importing the original as `requestFn`:

```ts
import { request as requestFn } from "./request";

const BASE = "https://api.example.com/v1";

function authHeaders() {
  const key = process.env.EXAMPLE_API_KEY;
  if (!key) throw new Error("EXAMPLE_API_KEY is not set");
  return { Authorization: `Bearer ${key}`, Accept: "application/json" };
}

const request = <T>(path: string, init?: RequestInit) =>
  requestFn<T>(`${BASE}${path}`, { headers: authHeaders(), ...init });

export const posts = {
  list: () => request<Post[]>("/posts"),
  get: (id: number) => request<Post>(`/posts/${id}`, { next: { revalidate: 3600 } }),
};
```

The module-level `request` is the only acceptable wrapper around `requestFn`. It must only bake in shared parameters like the base URL and auth headers

---

### File layout within a module

1. **Imports** — types first, then `request`/`requestFn` from `./request`
2. **Private config** — base URL, TTL constants, auth header helper
3. **Module-level `request` wrapper** one line, bakes in base URL and auth
4. **Namespace exports** — one `export const` per domain noun

---

## Grouping rules

- Group by **domain noun**: `movies`, `tv`, `subtitles`, `mappings`
- Name functions as **verbs on that noun**: `details`, `credits`, `search`, `list`, `create`, `remove`
- Avoid `get`/`fetch`/`load` prefixes — prefer `details` over `getDetails`

---

## The `request` utility

`request<T>(url, init?)` wraps `fetch`, calls `.json()`, and casts to `T`. All standard `RequestInit` options (`signal`, `method`, `headers`, `next`, …) pass through unchanged.

```ts
import { request } from "./request";
```

---

## Adding a new service

1. Create `services/<name>.service.ts`
2. Decide the category: client (imports `request` directly) or server (declares a module-level thin wrapper)
3. Add private config at the top — do not export it
4. Export one `const` per domain noun, each containing arrow functions
